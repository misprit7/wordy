Wordy Language Reference v0.1

The programming language where your source code is a Word document.

Overview

Wordy is an esoteric programming language where source code is written in Microsoft Word .docx files. Rather than using traditional text-based syntax, Wordy uses Word's rich formatting features — fonts, styles, tables, text alignment, and typographic effects — as its syntax.

Types are determined by font family. Functions are defined by headings. Control flow lives inside tables. Return statements are right-aligned paragraphs. Grouping expressions uses bold and highlighted text instead of parentheses. Italic text denotes string literals. Changing the font of a value casts it to a different type.

Philosophy
Every formatting feature in Microsoft Word maps to a programming concept. If you know how to use Word, you know how to write Wordy.

Font Matching
Script/cursive fonts include Brush Script MT, Lucida Handwriting, and Segoe Script. Any unrecognized font (including Calibri and Cambria Math) carries no type info — use these as neutral fonts for operators and code text. Variables and parameters must use a type font.

Functions

Functions are declared using Word heading styles (Heading 1, Heading 2, etc.). The heading text becomes the function name, and the heading's font determines the return type. If the heading uses a non-type font, the function returns void.

Factorial
A heading in Courier New with Heading 1 style
Declares function Factorial returning int

Entry Point (Main)

The program entry point is declared using a Drop Cap paragraph. In Word, a drop cap is a large initial letter at the start of a paragraph (Format → Drop Cap). The paragraph with a drop cap becomes the Main function.

The drop cap's initial letter is automatically stitched together with the continuation paragraph to form the first line of the entry point. This merged line is then parsed as a normal statement — it can be a variable assignment, function call, print, or any other valid statement.

Print factorial 10
Drop cap "P" + continuation "rint factorial 10"
Merged into print(factorial(10))

Parameters

Function parameters are declared in a Subtitle-styled paragraph immediately after the heading, or in a plain paragraph with typed-font runs. Each run's text is the parameter name and its font determines the type.

n   label
Subtitle paragraph with two runs
n: int, label: string

Parameter names are case-insensitive and stored in lowercase internally.

Return Values

A right-aligned paragraph is a return statement. The expression in the paragraph is evaluated and returned from the current function.

→ n × result
Right-aligned paragraph
Equivalent to return n * result;

Variables & Assignment

Assignment uses the leftward arrow symbol ← (Unicode U+2190). The first assignment to a name implicitly declares the variable.

Wordy result ← 1 ← declares & assigns (var result = 1;) result ← result × n ← reassigns (result = result * n;)

Identifiers are case-insensitive. They are sanitized to PascalCase in the generated C# output, with special characters replaced by underscores.

Variable References

Any identifier token that is not a keyword or followed by a bracket is treated as a variable reference. Names are lowercased internally.

Function Calls

A function call is an identifier followed by a formatting bracket containing the argument. Juxtaposition inside brackets also triggers call parsing.

Wordy factorial(n − 1) ← bold text around "n − 1" creates a grouping bracket

Formatting as Brackets

Instead of parentheses, Wordy uses text formatting changes to group expressions. When a formatting property is turned on, it acts as an opening bracket. When it's turned off, it acts as a closing bracket.

FormattingEffectBracket Type
BoldToggle on/offOpen/Close group
HighlightAny highlight color on/offOpen/Close group
Note
Italic text is not a bracket type — it is reserved for string literals.

Multiple formatting types can nest, enabling nested grouping:

a + b × c + d
Bold opens outer group, highlight opens inner group
a + (b * (c + d))
Careful
Formatting brackets are tracked via a stack. Each unique formatting flag (bold, highlight) that turns on pushes to the stack; turning off pops matching entries. Mismatched formatting may produce unexpected grouping.

String Literals (Italic Text)

Italic text is treated as a string literal. The raw text content is preserved exactly as written — operators and whitespace within italic runs are not parsed.

Hello World!
Italic text in Word
Equivalent to "Hello World!"
No Quotes Needed
Unlike most languages, Wordy doesn't use quotation marks for strings. Simply italicize text in Word to make it a string literal.

Type Casting

Changing the font of a variable or number expression casts it to the corresponding type. If a variable or number appears in a type font, a cast expression is generated.

SourceFont AppliedGenerated C#
A number 42Times New Roman42.ToString()
A variable xCourier New((int)(X))
A variable yScript/Cursive((double)(Y))
A variable nComic Sans MSConvert.ToBoolean(N)
A variable cSymbolConvert.ToChar(C)
Note
Numbers in numeric fonts (Courier New for Int, Script for Float) are not cast, since they are already the natural numeric type. Casting only applies when the font implies a different type than the expression's natural type.

Control Flow

All control flow in Wordy is expressed through Word tables. The structure and shape of the table determine which control flow construct it represents.

If Statements

An if statement is a table with a single merged cell on top (the condition) and two cells below (true and false branches).

If Statement — Table Layout
condition expression
true branch
statements...
false branch
statements...

This supports both 2-row format (condition row + branch row) and 3-row format (condition + patterns + bodies).

Generated C# if (condition) { // true branch } else { // false branch }

Match Statements

A match/switch statement uses the same merged-top-cell structure but with more than two cells in the rows below. Each cell represents a case.

Match Statement — Table Layout
subject expression
pattern 1
body...
pattern 2
body...
pattern 3
body...
(empty)
default...

Patterns can be comma-separated within a single cell to match multiple values (e.g. 3,6,9,12). A cell with the text _ (underscore) or no pattern becomes the default case.

Match statements also support a 3-row format with separate pattern and body rows for more complex layouts.

For Loops

A for loop is a table with three cells in the top row: initialization, condition, and step. The remaining rows form the loop body.

For Loop — Table Layout
init
i ← 1
condition
i <= 100
step
i ← i + 1
body
statements executed each iteration...
Generated C# for (var I = 1; I <= 100; I = I + 1) { // body statements }
Nested Control Flow
If the body rows of a for loop form a valid if/match table structure (single merged cell on top, multiple cells below), they are automatically parsed as nested control flow rather than sequential statements.

Print Statement

Calling print as a function prints the argument to standard output. Any call expression named "print" is automatically converted to a Console.WriteLine statement. This works anywhere — in the entry point, in regular functions, or inside control flow.

Wordy print Hello, World! ← bold bracket wraps the argument
